# Git Commit Message Generator

다음 파일에서 git 변경사항을 읽어서 Conventional Commits 형식의 커밋 메시지를 작성해줘:
**파일 경로:** `../scripts/git-changes.txt`

생성한 커밋 메시지는 메시지 경로에 txt 파일 형태로 저장하고 기존 파일 삭제하고 새로 생성해줘:

**메시지 경로** `../temp/commit-message.txt`

## 요구사항
1. **Conventional Commits 형식 사용**
   - feat: 새 기능 추가
   - fix: 버그 수정
   - docs: 문서 변경
   - style: 코드 포맷팅 (기능 변경 없음)
   - refactor: 리팩토링
   - chore: 빌드/설정 변경

2. **제목 작성 규칙**
   - 작업 내용의 핵심을 포괄하도록 문장 구성
   - 50자 이내
   - 한글로 작성
   - 명령형으로 작성 ("추가함" ❌, "추가" ⭕)
   - 마침표 없이

3. **본문 작성 규칙** (필요한 경우)
   - 무엇을, 왜 변경했는지 설명
   - 여러 변경사항이 있으면 항목별로 나열

4. **출력 형식**
```
<type>: <제목>

<본문 (선택사항)>
```

5. **주의사항**
   - 여러 관련 변경사항은 하나의 커밋으로 묶기
   - 관련 없는 변경사항이 섞여있다면 따로 커밋하도록 안내
   - 코드 설명보다는 **변경 이유와 목적**에 집중
